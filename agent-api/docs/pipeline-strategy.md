# 开店团队智能体流水线协作策略

## 概述

开店团队采用**串行+并行混合**的流水线协作模式，5个专业Agent按顺序执行，每个Agent的输出作为下一个Agent的输入，最终生成完整的协作报告。

## 执行流程

```
         选品专家
             ↓
         分析专家 ← 接收选品结果
             ↓
         财务顾问 ← 接收分析专家的结果（非选品结果）
             ↓
         决策引擎 ← 同时接收：① 分析专家的结果 ② 财务顾问的结果，一并对照处理
             ↓
         运营管家 ← 接收决策结果
             ↓
         汇总报告 ← 整合所有结果
```

## 各Agent职责与输入输出

| 步骤 | Agent | 输入 | 输出 |
|------|-------|------|------|
| 1 | 选品专家 | 选题主题（如"今日热销商品调研"） | 推荐若干热销商品 |
| 2 | 分析专家 | 选品专家的输出 | 市场分析报告 |
| 3 | 财务顾问 | **分析专家的输出**（不是选品结果） | 财务可行性评估 |
| 4 | 决策引擎 | 分析专家+财务顾问两个结果 | 最终3款商品选择 |
| 5 | 运营管家 | 决策引擎的输出 | 详细运营方案 |
| 6 | 汇总 | 所有步骤结果 | 完整协作报告 |

## 关键设计要点

### 决策引擎的输入
决策引擎需要**同时持有**分析专家和财务顾问两个结果进行对照：

```
请基于以下两份报告，做出最终的3款商品选择决策：

【分析专家报告】
{analysisResult}

【财务顾问报告】
{financialResult}

请综合考虑市场潜力、竞争程度、利润空间、风险因素，给出最终的3款推荐商品及理由。
```

### 财务顾问的输入
财务顾问接收的是**分析专家的结果**（不是选品结果），因为财务评估需要基于市场分析来做。

## 代码实现位置

- **PipelineService**: `app/Services/PipelineService.php`
- **PipelineRun命令**: `app/Console/Commands/PipelineRun.php`
- **ExecuteCronJob**: `app/Jobs/ExecuteCronJob.php`（支持`mode=pipeline`配置）

## 定时任务配置

在CronJob的config中设置`mode=pipeline`即可启用流水线模式：

```json
{
  "mode": "pipeline"
}
```

## 执行命令

```bash
php artisan pipeline:run --topic="今日热销商品调研"
```

## 已知问题

1. **超时问题**: qwen2.5:3b模型响应慢（单次10-30秒），完整流水线需要5-10分钟
2. **内存压力**: 同时运行多个Ollama模型会导致内存不足
3. **建议**: 生产环境考虑使用云端API或更强的本地硬件

## 执行日志

每个步骤的执行结果都会独立保存到`execution_logs`表，通过`pipeline_id`关联，可追溯完整流程。
